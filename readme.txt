//地图制作
使用Tiled工具来进行编辑，导出文件格式为.lua,并把对应的材质文件放入assets中的对应map里image的路径
目前只支持一张tileset的制作(记住选择嵌入地图)

地图文件中有一张地图gametest展示了全部功能，可以自行查看

特殊layer.class名：
--block 会自动把该layer的碰撞加载在碰撞系统(phisics.lua)中(只能有一个)

特殊object.class名：
--key 用于放一些重要的东西（下面有展示）
--normal 一些机关类，移动类的东西（下面有展示）

[key]:
type == "spawn" -> 玩家默认起始复活点，开始点
type == "checkpoint" -> 玩家hitbox接触后设置复活点
type == "setDash0","setDash1","setDash2", -> 玩家hitbox接触后设置默认Dash次数
type == "changeMap" -> 玩家hitbox接触后改变地图其对象名字为路径,起始路径为 "游戏目录.maps" + 路径
type == "refresh" --> 恢复水晶，将玩家的dash次数及体力恢复至最高，必须要有wait(int，ms单位)属性(大小强制规定12，12)

[normal]:
type == "spring" -> 弹簧（有方向属性int[上1,左2,下3,右4]，主要根据方向来设定大小，因为默认2x1大小，方便做图）
type == "mover_?" -> 默认移动块，里面有很多属性（必须），但改移动一般为一次性或者有回复功能，需要重复移动请自行添加对应属性
{
   --//必带
   bounce -> 固定时间回弹效果，可把玩家弹出去 具体规则看下面
   break -> 结束时自动破坏,要回复必须设置reStar，reTime
   canClimb -> 是否可以爬（不包括滑）
   getTime -> 如果moveX或者moveY不为0则为触发时移动到目的地的时间(ms)
   moveX,moveY -> 目前位置来移动的位置
   passive -> 被动触发 具体规则看下面
   reStar -> 是否启用回复 具体规则看下面
   reTime -> 回复时间(ms) 具体规则看下面
   repeat -> 来回移动 具体规则看下面
   shake -> 开始时的震动（视觉上） 具体规则看下面
   texture -> 材质(只能选择一种) 具体规则看下面
   wait -> 开始移动的停顿时间(ms) 具体规则看下面
   --//可选择（优先级更大）
   specific -> 特殊类型(11为上单向板（左挂），12为上单向板（右挂），2_为左单向板（上下挂同理），3_为下单向板（左右挂同理），4_为右单向板（上下挂同理）)
   multiTextrue -> string,数字+空格(可换行，当空格处理)，从左到右，从上到下依次选择对应的texture
}
--机关模式顺序（从上向下依次选择，即系统判断为该模式时直接使用对应条件）
1 -> repeat[打开repeat] : 机关来回移动，只读moveX,moveY,getTime,canClimb,wait(到达指定点后停顿时间),texture(对应地图tileID)
2 -> bounce[打开bounce] : 机关回弹,只读break(要回复必须设置reStar，reTime),getTime,canClimb,texture(对应地图tileID)
3 -> move[关闭repeat与bounce] : 机关单向移动,只读break(要回复必须设置reStar，reTime),canClimb,passive(打开后需要trigger_触发),
                               shake(是否在触发时摇动),wait(在触发后多久开始移动),texture(对应地图tileID)
--机关触发机制(trigger_)
非主动需要额外一个触发对象，type对应为trigger_ + ID
如果机关模式为move且type为mover_ + ID以及passive为打开状态则会
在玩家hitbox碰到trigger对象后触发。
主动触发则需要玩家脚下碰着或者抓/滑到墙自动触发

//地图背景(均为可选)
地图背景使用多视差效果展现出来
从远到近大概有4层
Sky层
Mount/Light层(以mount名字为准)
tree1层
tree2层
snow层
其中Sky层,Mount层,tree层需要在Tiled里的地图添加路径属性(string)，分别为skyPath,mountPath,treePath,其
默认目录为游戏main.lua所在目录下，推荐贴图都放在assets/Background里,如果不设置则默认没有。

其中snow层较为特殊，只能使用内置代码的预设，添加int属性名字为snowStyle，按规定数字来设定
当然你也可以自己在render.lua的对应位置写上你自己的效果
不设置 = 0
snowStyle = 1 : 上飘雪
snowStyle = 2 : 左飘雪
snowStyle = 3 : 下飘雪
snowStyle = 4 : 右飘雪
snowStyle = 5 : 下雨
snowR(int,255),snowG(int,255),snowB(int,255) : snow的颜色

//额外设置(不设置则使用默认)
blur(int) 渲染模糊度 推荐 30～60 (限制0～100)
r (int) 滤镜颜色R 0 ~ 255
g (int) 滤镜颜色G 0 ~ 255
b (int) 滤镜颜色B 0 ~ 255
RGBsize (int) 滤镜程度 0 ~ 100
music (string) 必须为以加载音频中以名字挑选！
